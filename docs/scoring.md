# 📊 记忆评分算法

## 重要性评分

```
importance = (base_weight × recency_factor × reference_boost) / 8.0
```

### 公式参数

| 参数 | 说明 | 计算方式 |
|------|------|----------|
| `base_weight` | 基础权重 | 1.0（普通）→ 2.0（HIGH）→ 1.0（PERMANENT=1.0） |
| `recency_factor` | 时效性 | `max(0.1, 1.0 - 天数/180)` |
| `reference_boost` | 引用加成 | `log₂(count + 1)` |

### 标记权重

| 标记 | 权重 |
|------|------|
| `⚠️ PERMANENT` | 1.0（固定） |
| `🔥 HIGH` | 2.0 |
| `📌 PIN` | 1.5 |
| 普通条目 | 1.0 |

## 健康度计算

```
health = (新鲜度×0.25 + 覆盖度×0.25 + 连贯度×0.2 + 效率×0.15 + 可达性×0.15) × 100
```

### 五维指标

| 指标 | 含义 | 计算 |
|------|------|------|
| **新鲜度** | 最近30天被引用的条目占比 | `活跃条目 / 总条目` |
| **覆盖度** | 最近14天有更新的知识类别 | `已更新类别 / 总类别` |
| **连贯度** | 至少有一个关联的条目占比 | `关联条目 / 总条目` |
| **效率** | MEMORY.md 精简程度 | `max(0, 100 - 行数/10)` |
| **可达性** | 记忆图谱连通程度 | `连通分量占比` |

## 遗忘曲线

条目必须满足以下条件才会被归档：
- >90 天未被引用
- 重要性 < 0.3
- 不是 PERMANENT 或 PIN

归档时保留原始 ID 和一行摘要。
