## 一、v1.x 遗留功能

### 1.1 完善报告模板 (P1)

**问题：当前报告太简单**

**苏格拉底式提问：**
- 什么样的报告对哥哥真正有价值？
- 报告应该回答什么问题？
- 如何让报告成为可执行的指南？

**设计：**

```markdown
## 🌀 梦境报告 — 2026-MM-DD

### 📊 统计
- 扫描: N 文件 | 新增: N | 更新: N

### ❤️ 健康度: XX/100 (↑/↓ 变化)
  ├── 新鲜度: 0.XX
  ├── 覆盖度: 0.XX
  ├── 连贯度: 0.XX
  ├── 效率: 0.XX
  └── 可达性: 0.XX

### 🔮 关键洞察 (最多 3 条)
1. **[类型]** 洞察描述
2. **[类型]** 洞察描述

### 📝 重要变更
- [mem_xxx] **新增** 标题 — 一句话描述
- [mem_xxx] **更新** 标题 — 变化点

### 💡 行动建议 (最多 2 条)
1. 建议内容 + 理由

### 🎯 下次 Dream 关注点
- 关注什么
```

### 1.2 实现增量备份 (P1)

**问题：当前全量备份太慢**

**苏格拉底式提问：**
- 备份的本质是什么？（数据安全）
- 什么时候需要恢复？（灾难场景）
- 全量和增量的区别是什么？

**设计：**

```
backup/
├── full-YYYYMMDD.tar.gz      # 每周全量
├── incr-YYYYMMDD.tar.gz     # 每日增量
└── manifest.json             # 备份清单
```

**策略：**
- 每周日：全量备份
- 其他天：增量备份（对比上周日的 base）
- 保留：最近 7 个增量 + 4 个全量
- 清单：记录每个备份的时间、内容、大小

### 1.3 健康度告警 (P1)

**问题：健康度下降时没有通知**

**苏格拉底式提问：**
- 什么程度的下降需要告警？
- 告警应该发送到哪？
- 告警频率如何控制？

**设计：**

```
健康度阈值：
├── 🔴 < 50: 严重告警（立即通知）
├── 🟡 < 70: 轻度告警（下次报告提及）
└── ✅ >= 70: 正常
```

## 二、v2.0 Generator-Evaluator 架构

### 2.1 三 Agent 系统

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    三 Agent 系统 (参考 Anthropic)                            │
└─────────────────────────────────────────────────────────────────────────────┘

  Supervisor (哥哥)
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ Planner Agent                                                           │
  │ • 分析哥哥的需求或 cron 触发                                            │
  │ • 决定 Dream 范围和重点                                                 │
  │ • 制定执行计划（不上下文）                                              │
  └─────────────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ Generator Agent (Dream)                                               │
  │ • 收集 (Collect)                                                       │
  │ • 整合 (Consolidate)                                                  │
  │ • 生成报告草稿                                                         │
  └─────────────────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │ Evaluator Agent                                                        │
  │ • 独立评审报告（避免自我偏袒）                                          │
  │ • 5 维度评分                                                          │
  │ • 输出具体改进建议                                                     │
  └─────────────────────────────────────────────────────────────────────────┘
         │       ↑
         └───────┘ 循环 (1-3 轮)
         │
         ▼
  推送最终报告
```

### 2.2 Evaluator 评审标准 (5维度)

| 维度 | 权重 | 问题 | 评分标准 |
|------|------|------|----------|
| 完整性 | 25% | 是否遗漏重要内容？ | 有无明显空白？ |
| 准确性 | 25% | 内容是否正确？ | 分类合理？ |
| 可读性 | 20% | 摘要清晰吗？ | 结构合理？ |
| 连贯性 | 15% | 与已有记忆矛盾？ | 关联正确？ |
| 可操作性 | 15% | 未来能检索吗？ | 有行动指引？ |

**决策规则：**
- 总分 < 60 → 返回 Generator 重做
- 总分 60-80 → 返回改进
- 总分 > 80 → 通过

### 2.3 Map not Manual 记忆分层

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Map not Manual (参考 OpenAI)                        │
└─────────────────────────────────────────────────────────────────────────────┘

  MEMORY.md (索引，≤100 行)
  ─────────────────────────────────────────────────────────────────────────
  ## [mem-xxx] 标题
  - 一句话摘要
  - 类型: fact/decision/lesson/procedure/person/project
  - 标签: #tag
  
  memory/
  ─────────────────────────────────────────────────────────────────────────
  ├── lessons/       # 教训详情（长文本）
  ├── decisions/    # 决策详情（上下文+理由）
  ├── people/       # 人物详情
  ├── procedures/   # 流程详情
  └── projects/     # 项目详情

  Context = 稀缺资源
  给 Agent 地图，不给说明书
```

### 2.4 Golden Principles

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Golden Principles                                   │
└─────────────────────────────────────────────────────────────────────────────┘

  不可违反的规则：
  
  1. 永不删除原始日志
  2. 永不移除 ⚠️ PERMANENT 标记
  3. 变更超过 30% 必须备份
  4. Evaluator 评分 < 60 分必须重做
  5. Context 超过 50% 必须压缩
```
